# User Journey

## Purpose

This document describes how a user interacts with Nuvio from the first visit to becoming job-ready.

---

# Journey

## Step 1 — Discover Nuvio

The user finds Nuvio through social media, GitHub, search engines, or a friend's recommendation.

Goal:
Understand what Nuvio is.

---

## Step 2 — Landing Page

The landing page explains:

* The problem
* Our mission
* Why Nuvio exists

The user clicks:

**Get Started**

---

## Step 3 — Sign Up

The user creates an account.

Required information:

* Name
* Email
* Password

---

## Step 4 — Onboarding

Nuvio asks a few simple questions.

Examples:

* What career do you want?
* Have you coded before?
* How much time can you study each day?

---

## Step 5 — Choose Career Goal

Available options (MVP):

* Frontend Developer
* Backend Developer
* MERN Stack Developer

---

## Step 6 — Skill Assessment

The user answers questions to evaluate current skills.

Topics include:

* HTML
* CSS
* JavaScript
* Git
* React
* Node.js
* Express
* MongoDB

---

## Step 7 — Skill Report

Nuvio shows:

* Overall score
* Strengths
* Weaknesses
* Missing skills

---

## Step 8 — Personalized Roadmap

Based on the assessment, Nuvio creates a personalized learning path.

The roadmap includes:

* Topics
* Projects
* Weekly goals

---

## Step 9 — Dashboard

The dashboard shows:

* Today's mission
* Progress
* Roadmap
* Job readiness score

---

## Step 10 — Daily Learning

Every day the user completes one clear task.

Examples:

* Learn React Hooks
* Build a Weather App
* Practice Git Branching

---

## Step 11 — Progress Tracking

Nuvio tracks:

* Skills completed
* Projects built
* Learning streak
* Progress toward becoming job-ready

---

## Step 12 — Job Ready

The user completes the roadmap and is prepared for internships or junior developer opportunities.

---

# Guiding Principle

Every screen in Nuvio must answer one question:

**What should I do next?**
